# Product Requirement Document (PRD): Event Registration System

## 1. Product Overview
This product is an "Event Registration System" designed for campus environments. It provides a platform for organizers to create and manage events, and for students to browse and register for them.

## 2. Target Audience
* **Organizers / Student Club Officers**: Need to publish events, manage registration lists, and track status.
* **Students**: Need to browse event info, register, and view details.

## 3. Roles & Permissions
* **General Users (Students)**: Browse events, view details, register.
* **Administrators (Organizers)**: All user features, plus create, edit, delete events, and view registration counts/lists.

## 4. Core Features
### 4.1 Event Management (Admin)
1. Create events.
2. Edit event info.
3. Delete events.
### 4.2 Event Display (Public)
4. Event list.
5. Event details.
6. Display registration counts.
### 4.3 Registration
7. Register for events (with capacity check).

## 5. Non-Functional Requirements
* Responsive Web Design (RWD).
* Modern UI/UX.
* Performance & Stability.

## 6. Future Roadmap
* QR Code check-in.
* List export (Excel/CSV).
* Waiting list.
* Auto notifications.

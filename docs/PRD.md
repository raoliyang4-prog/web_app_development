# 產品需求文件 (PRD)：活動報名系統

## 1. 專案概述
- **背景與動機**：為了有效管理學校或機構舉辦的各類活動，並簡化學生的報名流程，本專案旨在建立一個穩定且高併發的線上活動報名系統。
- **目標用戶**：
  1. **學生 / 一般使用者**：希望能快速瀏覽、查詢活動並順利完成報名。
  2. **系統管理員**：需要發布、編輯、刪除活動，並管理報名名單。
- **核心價值主張**：提供一個不超賣、穩定、直覺的操作平台，確保每次搶票與報名活動皆能順利完成。

## 2. 功能需求

1. **使用者身分與權限管理**
   - *使用者故事*：作為系統，我需要區分「管理者」與「學生」，確保不同角色只能操作對應的功能。
2. **活動管理 (CRUD)**
   - *使用者故事*：作為管理者，我希望可以建立、編輯與下架活動，並設定報名人數上限。
3. **活動瀏覽與搜尋**
   - *使用者故事*：作為學生，我希望可以瀏覽公開的活動列表，並查看詳細活動資訊與剩餘名額。
4. **活動報名機制**
   - *使用者故事*：作為學生，我希望可以在名額未滿時點擊報名，並確保系統能正確記錄我的報名狀態。
5. **報名名單與數據追蹤**
   - *使用者故事*：作為管理者，我希望可以匯出特定活動的報名名單，並隨時掌握報名人數。

## 3. 非功能需求
- **高併發防護**：系統必須能在瞬間大量報名請求下（搶票情境）保持資料一致性，嚴防「超賣」現象。
- **資料安全**：使用安全的驗證機制（如 JWT）保護 API 介面，並防範 SQL Injection 與 XSS 攻擊。
- **效能指標**：API 回應時間應盡可能短，確保使用者體驗流暢。

## 4. MVP 範圍

### Must Have (必備功能 - 初期實作)
- 基本的登入機制（學生/管理者角色）。
- 管理員建立、編輯活動（包含人數上限設定）。
- 學生瀏覽清單與進行活動報名。
- 後端報名的 Transaction 與 Row-level Lock 併發防護。

### Should Have (應有功能 - 中期實作)
- 活動關鍵字搜尋功能。
- 報名狀態查詢（報名成功、已取消等）。
- 管理者檢視個別活動的完整報名名單。

### Nice to Have (加分功能 - 未來擴充)
- 候補機制：當名額額滿時允許學生排入候補名單。
- 電子郵件/系統通知：報名成功後自動寄發確認信。
